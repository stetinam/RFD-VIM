# RFD-VIM (RFDiffusion Visual Input Manager)
> **9-2025_RFD-VIM:** Update with improved menu layouts and new functions: define a pocket region and generate reference structures for RMSD calculations.

Interactive PyMOL tool for visually selecting protein residues to freeze in RFdiffusion and ProteinMPNN workflows. A faster, more intuitive, and human error-free alternative to manually examining protein structures and typing out RFdiffusion contig settings. Streamlines input generation for RFdiffusion, ProteinMPNN, and RMSD calculations in your protein design pipelines.

## Features

- **Visual residue selection** - Click residues in PyMOL to freeze amino acid backbone or type
- **Pocket residue marking** - Define vital pocket (or just important for your function) residues
- **Real-time feedback** - Instant visual updates with color coding, see your settings in real time
- **Enhanced PyMOL sessions** - Save .pse files with RMSD reference objects for RMSD calculations
- **Export ready** - Generates CONTIGS, INPAINT_SEQ, and POCKET_RESIDUES for RFdiffusion and ProteinMPNN workflows

> **Note:** Exports only the frozen contigs, does not output segments RFdiffusion should generate, must be added by user later

## Usage

### Quick Start
```bash
python rfd-vim.py
```

### Workflow
1. **Load PDB** → Type `file filename.pdb` or `fetch PDB_ID` directly in PyMOL
2. **Choose initial settings** → Load from saved file, script, or start empty
3. **Interactive editing** → Click residues in PyMOL and set freeze status
4. **Save settings** → Export CONTIGS, INPAINT_SEQ, and POCKET_RESIDUES
5. **Save PyMOL session** → Save .pse with RMSD reference objects

### PyMOL Commands
> **Note:** Type all commands directly into PyMOL
- **Structure loading**: `file filename.pdb`, `fetch PDB_ID`
- **Menu options**: `1`, `2`, `3`, `4`, `5`, `6`
- **File operations**: `file {filename.txt}` (replace filename with actual path)
- **Residue editing**: `bt`, `b`, `p`, `n`, `q`

### Residue States
- **bt** = Backbone + Type of amino acid frozen (green sticks) - Critical residues
- **b** = Backbone only frozen (orange lines) - Structurally important, sequence flexible
- **p** = Pocket residue (dark green sticks) - Mark as pocket/critical residue for RMSD calculations (optional)
- **n** = Not frozen (cyan cartoon) - Regions not preserved in designs generated by RFdiffusion
- **q** = Exit editing mode

### Edit Workflow
1. Select residues in PyMOL (click in the structure, select in sequence or use a command, see below)
2. Type residue state in PyMOL (`bt`, `b`, `p`, or `n`)
3. Changes apply instantly
4. Type `q` to finish editing

> **Note:** You may select multiple residues with a PyMOL command, but it must be in following format

`select sele, protein & (resi 5,10,53-52)`
- change the numbers to the positions you actually want to select and then change freeze status

## Output Format: ready for RFdiffusion
```
CONTIGS="A2-15/A50-88/A131-147"
INPAINT_SEQ="A50-56/A59-59/A61-62"
POCKET_RESIDUES="A25-30/A75-80" (optional)
```

- **CONTIGS** → All preserved residues (bt + b states)
- **INPAINT_SEQ** → Backbone-only preserved residues (b state ONLY)
- **POCKET_RESIDUES** → Pocket residues (p command) (optional)

> **Note:** This means bt state residues are in CONTIGS and NOT in INPAINT_SEQ

## PyMOL Session Features

The PSE Edition includes enhanced PyMOL session saving with automatic RMSD reference object creation:

### RMSD Reference Objects
When saving a .pse file, the following reference objects are automatically created:

- **pocket_RMSD_ref** - Contains pocket residues for critical residue RMSD calculations (optional)
- **MPNN_RMSD_ref** - Contains fully frozen residues (bt state) for MPNN comparison
- **RFdiff_RMSD_ref** - Contains all frozen residues (bt + b states) for RFdiffusion RMSD

### Usage
> **Note:** Type all commands directly into PyMOL
1. Start the program and load a protein structure (file or fetch from PDB)
2. Choose initial settings (load from saved file, script file, or start empty)
3. Use interactive editing mode to set residue states (bt, b, p, n)
4. Save your settings as .txt file (option 4) - contains CONTIGS, INPAINT_SEQ, POCKET_RESIDUES
5. Save PyMOL session as .pse file (option 5) - contains structure, visual states, and RMSD reference objects (optional)
6. Use saved files for protein generation and RMSD analysis

## Menu Structure

### Initial Setup Menu

- **'6'** → Exit at any menu
- **'q'** → Exit editing mode

## Example Walkthrough

### Step-by-Step Example with ASNase.pdb

After cloning the repository (see Setup above), you can try the example workflow:

1. **Start the program**:
   ```bash
   python rfd-vim.py
   ```

2. **Load the example protein structure**:
   - When prompted, type: `file ASNase.pdb` directly into PyMOL
   - This will load the structure of an asparaginase (an enzyme that cleaves asparagine) into PyMOL

3. **Load the example configuration**:
   - Choose option `1` (Load from saved file)
   - Type: `file asnase_example.txt`
   - This will load predefined freeze settings with:
     - Green sticks: Fully frozen residues (backbone + type)
     - Orange lines: Backbone-only frozen residues
     - Cyan cartoon: Flexible regions

4. **Interactive editing**:
   - Choose option `1` (Interactive editing)
   - Click on residues in PyMOL to select them
   - Type commands directly in PyMOL command line:
     - `bt` = Backbone + Type frozen (green sticks)
     - `b` = Backbone only frozen (orange lines)  
     - `p` = Pocket residue (dark green sticks)
     - `n` = Not frozen (cyan cartoon)
     - `q` = Quit editing mode

5. **Save your modifications**:
   - Return to main menu (`q`) and choose option `4` (Save settings)
   - Type: `file asnase_my_example.txt`
   - This saves your customized CONTIGS, INPAINT_SEQ, and POCKET_RESIDUES settings

6. **Save PyMOL session**:
   - Choose option `5` (Save .pse file for RMSD calculations)
   - Type: `file asnase_session.pse`
   - This saves the full PyMOL session with RMSD reference objects

### PyMOL Commands Reference
- **Structure loading**: Type `file filename.pdb` or `fetch PDB_ID` in PyMOL
- **Menu navigation**: Type `1`, `2`, `3`, `4`, `5`, or `6` in PyMOL
- **File operations**: Type `file filename.txt` in PyMOL 
- **Residue editing**: Type `bt`, `b`, `p`, `n`, or `q` in PyMOL
